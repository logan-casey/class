I provide simplified code below for reference -- the full code includes many options to dampen updates, improve performance, and record diagnostics. I omit code to set parameters, build grids, discretize the AR(1) shock, calculate the default threshold and the feasible borrowing range, and the code to run simulations.

The procedure is simple. Given a candidate interest rate schedule $\tilde{r}$, \texttt{solve\_vfi.m} iterates the Bellman equation given by \texttt{bellman\_rhs\_fast.m} (eq 13 in HW) to convergence. For the resulting value and policy functions, \texttt{update\_rtilde.m} provides the implied interest rate schedule (eq 20 in HW). \texttt{solve\_equilibrium.m} loops over these two steps until a final solution is achieved. \texttt{main.m} sets up the parameters and grids and runs the whole process.
